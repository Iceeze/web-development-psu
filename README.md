# Лабораторные работы по курсу «Разработка Web-приложений»

## Лабораторная работа №1 (КТ1, 15 баллов)

**HTML и CSS, верстка сайтов (повторение)**

Разработать дизайн и общую структуру сайта любой медицинской организации со следующей структурой:

- Новости и акции (3-4 блока),
- Наши услуги (использовать маркированный или нумерованный список),
- Наши специалисты (использовать карточки с указанием фото, ФИО и специальности врача), обязательно использование либо flex, либо grid-верстки
- Расписание работы специалистов (обязательно использование таблицы)
- Контакты (использовать скрипт для геолокации).

<u>Требования к сайту:</u>

- Обязательно наличие «шапки» сайта с названием медицинской организации и логотипом
- Использование единого стиля с применением CSS.
- Наличие панели навигации на всех страницах сайта.
- Обязательно использование либо flex-верстки, либо grid-верстки для создания макета сайта
- Использование адаптивной верстки

## Лабораторная работа №2 (КТ1, 15 баллов)

**JavaScript**

Создать Web-приложение «Калькулятор», позволяющее переводить любое заданное целое положительное число из одной системы счисления в другую (все системы счисления от 2 до 16!). Для выбора систем счисления использовать списки (один - из какой системы переводить, другой - в какую систему), для ввода чисел использовать **КНОПКИ**! То есть, должно быть 16 кнопок с числами и плюс две кнопки для вывода результата и сброса числа.

Вывод результата осуществлять в текстовое поле (поле должно быть недоступно для ввода!). После выбора системы счисления недопустимые в этой системе счисления кнопки должны стать неактивными во избежание случайного нажатия.

## Лабораторная работа №3 (КТ2, 10 баллов)

**Язык PHP, базы данных**

Переработать сайт из задания 1, используя возможности языка PHP:

1.  Информация на 1 и 3 странице должна браться из базы данных
2.  Добавить на сайт страницу с формой записи к врачу. После отправки формы информация должна заноситься в базу данных
3.  Создать шаблоны для повторяющихся элементов страницы («шапка» сайта, нижний футер, панель навигации)
4.  На главной странице должна выводиться текущая дата и день недели (для реализации использовать PHP)

## Лабораторная работа №4 (КТ2, 10 баллов)

**Парсинг**

Разработать ресурс, который позволяет с помощью средств парсинга получить информацию с любого сайта (например, информацию о товарах двухкомнатных квартирах в Индустриальном районе или информацию об автомобилях Geely Cityray выпуска 2023 года и т.п. на avito.ru). Для промежуточного хранения данных можно использовать файлы или БД. Допускается выполнить парсинг с занесением информации в БД или файл с помощью средств языка Python, а затем вывести ее в виде HTML-документа с использованием php. Либо все задание выполнить с помощью одного любого языка или фреймворка.

## Лабораторная работа №5 (КТ2, 10 баллов)

**Использование API**

Разработать ресурс, который позволяет отображать информацию определенного вида (например, информацию о текущей погоде или о результатах футбольных матчей), полученную с помощью API. Отображать информацию в виде таблицы с количеством строк не менее 20 и количеством столбцов не менее 5.

## Лабораторная работа №6 (КТ3, 40 баллов)

**Итоговое задание**

Разработать сайт на произвольную тему, используя любой фреймворк (Laravel, Node.js, Django и т.д.). 
<u>Требования к сайту</u>:

- сайт должен содержать пользовательскую часть и раздел администрирования,
- количество страниц пользовательской части -- не менее трех,
- вся информация на сайте должна браться из базы данных,
- сайт должен быть адаптирован под мобильные устройства,
- обязательно использовать либо flex, либо grid верстку для создания макета сайта,
- вход в раздел администрирования должен осуществляться по логину и паролю,
- раздел администрирования должен позволять редактировать всю информацию на сайте (добавление, редактирование и удаление записей),
- сайт должен иметь осмысленное и хорошее наполнение (бессмысленное содержание оцениваться не будет),
- использовать на сайте любые два дополнительных элемента (слайдеры, календарь, информер погоды, информер валют и т.д.)
- запрещено в качестве темы сайта использовать темы, связанные с ПГНИУ!
